# Этапы разработки MeetMaster v1

## Назначение документа

Этот документ раскладывает техническое задание MeetMaster на этапы разработки.

Для каждого этапа указано:

- что должно быть сделано;
- что разработчик обязан проверить сам;
- что нужно проверить владельцу проекта;
- как проверить результат без знания кода;
- критерии приемки этапа.

Главное правило: исполнитель должен максимально проверять работу самостоятельно. Владелец проекта подключается только там, где без него нельзя проверить доступы, реальные уведомления, Google-аккаунт или удобство пользовательского сценария.

---

## Общие правила приемки

| Правило | Описание |
|---|---|
| Исходный код | хранится в репозитории |
| Секреты | токены, пароли, ключи и данные доступа не хранятся в коде |
| Проверка | каждый этап завершается демонстрацией результата |
| Ошибки | если найден дефект, этап не считается завершенным |
| Документация | после каждого этапа обновляются инструкции, если изменилась настройка или запуск |
| Язык интерфейса | русский |
| Часовой пояс | Москва |

Секреты нельзя отправлять в чат. Токен Telegram-бота, пароль от сервера, ключи Google и данные базы должны передаваться только безопасным способом или вноситься владельцем проекта самостоятельно по инструкции.

## Итоговый порядок разработки

| Этап | Название | Результат |
|---:|---|---|
| 0 | Подготовка доступов и окружения | есть бот, сервер, Google API, база |
| 1 | Каркас проекта и запуск бота | бот отвечает в Telegram |
| 2 | База данных и хранение заявок | данные сохраняются |
| 3 | Пользовательский сценарий создания заявки | пользователь создает заявку |
| 4 | Расчет свободных слотов | показываются только доступные слоты |
| 5 | Интеграция с Google Календарем | встречи создаются в календаре |
| 6 | Админка в Telegram | администратор управляет заявками |
| 7 | Уведомления и напоминания | стороны получают сообщения |
| 8 | Отмена и перенос встреч | встречи можно менять и отменять |
| 9 | Логи, бэкапы и мониторинг | бот готов к эксплуатации |
| 10 | Финальное тестирование MVP | весь сценарий проверен |
| 11 | Развертывание на сервере | бот работает постоянно |
| 12 | Передача результата и инструкция | владелец проекта может пользоваться MVP |

## Что точно нужно от владельца проекта

| Что нужно | Когда нужно | Комментарий |
|---|---|---|
| Telegram-бот через BotFather | этап 0 | токен не отправлять в обычный чат |
