Дата актуализации: 2026-05-14

## 1. Что уже сделано

Бот MeetMaster размещен на VPS:

```text
IP: 155.212.135.186
Пользователь: root
ОС: Ubuntu 24.04.4 LTS
Папка проекта: /opt/meetmaster/app
```

Бот запущен через Docker Compose:

```text
app-bot-1  - контейнер Telegram-бота

```

На сервере уже были другие сервисы:

```text

/opt/stack/n8n
```

Их нельзя удалять или переустанавливать без отдельной проверки.

## 2. Важное предупреждение

На сервере мало свободного места. После размещения бота диск был занят примерно на `93%`, свободно было около `1.8-2.1 ГБ`.

Перед любыми крупными обновлениями сначала проверить:

```bash
df -h /
docker system df
du -h --max-depth=1 /var/lib/docker 2>/dev/null | sort -h
```

Нельзя вслепую выполнять:

```bash
docker system prune -a
rm -rf /opt/stack
rm -rf /var/lib/docker
```

Эти команды могут удалить Supabase, n8n или их данные.

## 6. Секретные файлы

Файлы с секретами лежат только на локальном компьютере и на сервере. Их нельзя отправлять в чат или GitHub.

На сервере:

```text
/opt/meetmaster/app/.env
/opt/meetmaster/app/credentials/google_client_secret.json
/opt/meetmaster/app/credentials/google_token.json
```

Права на сервере:

```bash
chmod 600 /opt/meetmaster/app/.env
chmod 700 /opt/meetmaster/app/credentials
chmod 600 /opt/meetmaster/app/credentials/google_client_secret.json
chmod 600 /opt/meetmaster/app/credentials/google_token.json
```

В `.env` должны быть переменные:

```text
BOT_TOKEN
ADMIN_TELEGRAM_IDS
DATABASE_URL
GOOGLE_CALENDAR_ENABLED
GOOGLE_CALENDAR_ID
GOOGLE_OAUTH_CLIENT_SECRET_FILE
GOOGLE_OAUTH_TOKEN_FILE
TIMEZONE
WORKDAY_START
WORKDAY_END
MIN_NOTICE_DAYS
BUFFER_MINUTES
ALLOWED_DURATIONS
LOG_LEVEL
POSTGRES_USER
POSTGRES_DB
POSTGRES_PASSWORD
```

`POSTGRES_PASSWORD` был сгенерирован на сервере и не должен храниться в открытом виде в документах.

## 7. Первичное размещение на сервере

Создать папку:

```bash
mkdir -p /opt/meetmaster
cd /opt/meetmaster
```

Склонировать репозиторий:

```bash
git clone https://github.com/swet757-cpu/meetmaster.git app
cd app
```

С локального компьютера скопировать `.env`:

```powershell
scp -i .deploy_keys\meetmaster_beget_ed25519 .env root@155.212.135.186:/opt/meetmaster/app/.env
```

Создать папку для Google credentials:

```bash
mkdir -p /opt/meetmaster/app/credentials
chmod 700 /opt/meetmaster/app/credentials
```

С локального компьютера скопировать Google credentials:

```powershell
scp -i .deploy_keys\meetmaster_beget_ed25519 credentials\google_client_secret.json root@155.212.135.186:/opt/meetmaster/app/credentials/google_client_secret.json
scp -i .deploy_keys\meetmaster_beget_ed25519 credentials\google_token.json root@155.212.135.186:/opt/meetmaster/app/credentials/google_token.json
```

На сервере добавить переменные PostgreSQL, если их еще нет:

```bash
cd /opt/meetmaster/app

if ! grep -q '^POSTGRES_PASSWORD=' .env; then
  {
    echo ''
    echo 'POSTGRES_USER=meetmaster'
    echo 'POSTGRES_DB=meetmaster'
    printf 'POSTGRES_PASSWORD='
    openssl rand -base64 24
  } >> .env
fi
```

Выставить права:

```bash
chmod 600 .env
chmod 600 credentials/google_client_secret.json
chmod 600 credentials/google_token.json
```

Запустить бота:

```bash
docker compose up -d --build
```

## 8. Что делает запуск

`docker compose up -d --build`:

1. Собирает образ Python-приложения.
2. Поднимает PostgreSQL `app-db-1`.
3. Ждет, пока база станет healthy.
4. Запускает контейнер `app-bot-1`.
5. Выполняет Alembic-миграции через `docker-entrypoint.sh`.
6. Запускает Telegram polling.

Проверить контейнеры:


Ожидаемо:




Проверить версию миграции:

```bash
printf '%s\n' "select version_num from alembic_version" | docker compose exec -T db psql -U meetmaster -d meetmaster
```

На 2026-05-14 актуальная версия:

```text
20260514_0002
```

Проверить тип Telegram ID:

```bash
printf '%s\n' "select column_name, data_type from information_schema.columns where table_name='users' and column_name='telegram_id'" | docker compose exec -T db psql -U meetmaster -d meetmaster
```

Ожидаемо:

```text
telegram_id | bigint
```

2. Выбрать дату.
3. Выбрать длительность.

```bash
cd /opt/meetmaster/app

```

## 12. Уже исправленная ошибка с Telegram ID

Была ошибка:

```text
invalid input for query argument ... value out of int32 range
WHERE users.telegram_id = $1::INTEGER
```

Причина: Telegram ID пользователя был больше, чем помещается в PostgreSQL `integer`.

Исправление:

```text
users.telegram_id теперь BigInteger / bigint
миграция: 20260514_0002_users_telegram_id_bigint.py
```

Проверка после исправления:

```text
alembic_version = 20260514_0002
users.telegram_id = bigint
```
