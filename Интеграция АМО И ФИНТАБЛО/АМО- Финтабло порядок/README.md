# AMO -> n8n -> Финтабло

Эта папка про интеграцию сделок из AMO в Финтабло через серверный n8n.

Рабочий вариант:

```text
AMO -> серверный n8n -> Финтабло
```

Cloud n8n для этой интеграции не использовать.

## Что должно получиться

Когда сделка в AMO попадает в статус "Успешно реализовано", в Финтабло создается сделка.

Одна сделка AMO должна создать только одну сделку Финтабло.

## Текущий рабочий итог

| Где | Значение |
|---|---:|
| AMO | 6 сделок |
| Финтабло | 6 сделок |
| Сумма AMO | 1 135 000 руб. |
| Сумма Финтабло | 1 135 000 руб. |

## Что должно быть включено

В серверном n8n должен быть опубликован workflow:

```text
amoCRM -> Финтабло: завершенные сделки
```

Схема внутри:

```text
Webhook
  -> If
  -> Проверить и запомнить ID сделки AMO
  -> HTTP Request
```

## Что проверять в n8n

Открой:

```text
https://n8n1.sweevottak.pro
```

Проверь:

| Что | Как должно быть |
|---|---|
| Workflow | `amoCRM -> Финтабло: завершенные сделки` |
| Статус | опубликован |
| Webhook | POST |
| If | проверяет `status_id = 142` |
| HTTP Request | отправляет в `api.fintablo.ru` |

В узле `If` слева должно быть ровно:

```text
{{ $json["body"]["leads[update][0][status_id]"] }}
```

Справа:

```text
142
```

Лишних пустых строк в формуле быть не должно.

## Что проверить в AMO

В AMO должен быть webhook на серверный n8n:

```text
https://n8n1.sweevottak.pro/webhook/amocrm-fintablo-deal
```

Проверить:

| Что | Как должно быть |
|---|---|
| Webhook AMO | один рабочий webhook |
| URL | серверный n8n1 |
| Cloud URL | не использовать |
| Событие | изменение сделки |

Если одновременно включены cloud n8n и серверный n8n, могут появиться дубли.

## Как проверить, что интеграция работает

1. В AMO создай тестовую сделку.
2. Укажи сумму.
3. Переведи сделку в статус "Успешно реализовано".
4. Открой серверный n8n.
5. Перейди в `Казни`.
6. Последний запуск должен быть успешным.
7. В запуске цепочка должна дойти до `HTTP Request`.
8. Открой Финтабло.
9. Обнови страницу.
10. Сбрось фильтры.
11. Найди сделку по названию или комментарию.

Комментарий в Финтабло:

```text
Создано из amoCRM. ID сделки: <ID сделки AMO>
```

## Если сделка не появилась в Финтабло

Проверяй по порядку:

| Шаг | Что смотреть |
|---|---|
| 1 | Пришел ли запуск в n8n |
| 2 | Дошел ли запуск до `If` |
| 3 | Ушел ли `If` в True Branch |
| 4 | Дошел ли запуск до `HTTP Request` |
| 5 | Вернул ли Финтабло `status: 200` |
| 6 | Нет ли фильтра в Финтабло |

Если запуск дошел только до `Webhook -> If`, значит проблема в условии.

Если запуск дошел до `HTTP Request`, значит n8n отправил сделку в Финтабло. Тогда искать в Финтабло по комментарию с ID сделки AMO.

## Если появились дубли

Проверь:

| Возможная причина | Что сделать |
|---|---|
| В AMO два webhook | оставить только серверный n8n |
| Cloud workflow еще активен | выключить cloud workflow |
| Антидубль очищен | восстановить ID в памяти антидубля |
| Тест повторно отправляли вручную | проверить комментарий с AMO ID |

Не удаляй дубли сразу. Сначала выпиши:

| AMO ID | Количество записей в Финтабло | Сумма |
|---|---:|---:|

Потом удалять только подтвержденные лишние записи.

## Как повторить такую интеграцию заново

1. Поднять серверный n8n.
2. Создать workflow.
3. Добавить `Webhook`.
4. Скопировать production URL webhook.
5. В AMO добавить webhook на этот URL.
6. Добавить узел `If`.
7. В `If` поставить проверку статуса:

```text
{{ $json["body"]["leads[update][0][status_id]"] }}
```

равно:

```text
142
```

8. Добавить узел `Code` для антидубля по ID сделки AMO.
9. Добавить `HTTP Request` в Финтабло.
10. В тело запроса передать:

| Поле Финтабло | Поле AMO |
|---|---|
| name | `leads[update][0][name]` |
| amount | `leads[update][0][price]` |
| currency | `RUB` |
| comment | `Создано из amoCRM. ID сделки: ...` |

11. Опубликовать workflow.
12. Сделать тестовую сделку.
13. Проверить AMO и Финтабло по количеству и сумме.

## Токены

Токены не вставлять в чат.

Токен Финтабло лучше хранить в n8n Credentials, а не прямо в HTTP Request.

Если токен случайно показали в чате или логе, его надо сменить.

## Когда ничего не трогать

Ничего не менять, если:

| Проверка | Результат |
|---|---|
| AMO | количество и сумма совпадают |
| Финтабло | количество и сумма совпадают |
| n8n | workflow опубликован |
| Последние казни | successful |

В этом состоянии интеграция работает.
