# Telegram-бот для аудита ОСВ и маппинга P&L

Бот принимает два файла:

| Файл | Форматы | Что делает |
|---|---|---|
| ОСВ из 1С | `.xlsx`, `.xls`, `.csv` | Ищет счет 20, оборот Дт, конечное сальдо Дт и аномалии |
| Маппинг_PL | `.xlsx`, `.xls`, `.csv` | Проверяет активность счета 20 и при необходимости отключает 90.02 |

Поддержан формат ОСВ из образца: шапка 1С в строках 1-7, данные с 8-й строки.

Поддержан формат маппинга со скриншота: поля `Активно`, `Статья`, `Счет mask`, `Источник`, `Знак`, `Сумма` стоят вертикально в первом столбце, данные начинаются ниже.

## Запуск

Создайте переменную окружения:

```powershell
$env:BOT_TOKEN="токен_бота"
```

Или создайте файл `.env` по образцу `.env.example`.

Установите зависимости:

```powershell
pip install -r requirements.txt
```

Запустите бота:

```powershell
python main.py
```

Можно одной командой через PowerShell:

```powershell
.\run_bot.ps1
```

Проверка логики без Telegram:

```powershell
python scripts\smoke_check.py
```

Проверка связи с Telegram API:

```powershell
.\.venv\Scripts\python.exe scripts\check_telegram_connection.py
```

## Режимы

| Режим | Результат |
|---|---|
| Аудитор | Только текстовые алерты |
| Авто-корректор | Текстовые алерты и копия исправленного маппинга |

Исходные файлы не перезаписываются. Исправленный маппинг сохраняется отдельной копией с суффиксом `_v2`.
