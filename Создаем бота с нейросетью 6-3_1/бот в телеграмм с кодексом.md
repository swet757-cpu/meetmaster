# Бот в Telegram с Codex

Пошаговая инструкция, как создать Telegram-бота как в этом проекте: бот принимает ОСВ и маппинг P&L, проверяет данные и может делать исправленную копию маппинга.

## 1. Что понадобится

| Что | Для чего |
|---|---|
| Telegram | создать бота через BotFather |
| Python 3.10+ | запуск кода бота |
| VS Code или папка проекта | хранить файлы проекта |
| Codex | написать и проверять код |
| VPN или прокси | если Python не подключается к Telegram API |

## 2. Создать бота в Telegram

1. Откройте Telegram.
2. Найдите `@BotFather`.
3. Напишите команду:

```text
/newbot
```

4. Введите название бота, например:

```text
Аудитор_финансиста
```

5. Введите username бота. Он должен заканчиваться на `bot`, например:

```text
PLAuditBot
```

6. BotFather выдаст токен.

Токен нельзя отправлять в чат, скриншоты, документы и логи. Если токен случайно раскрыт, его нужно сменить через BotFather.

## 3. Создать папку проекта

Создайте отдельную папку, например:

```text
C:\Users\Анна\Desktop\мой_telegram_бот
```

Откройте эту папку в VS Code или терминале.

## 4. Попросить Codex создать проект

В Codex напишите задачу:

```text
Сделай Telegram-бота на Python 3.10+ и aiogram 3.x.

Бот должен:
- работать через FSM;
- принимать файл ОСВ из 1С в формате xlsx/xls/csv;
- принимать файл Маппинг_PL в формате xlsx/xls/csv;
- иметь режимы Аудитор и Авто-корректор;
- проверять счет 20;
- если по счету 20 есть оборот Дт и он отключен в маппинге, показывать алерт;
- в авто-корректоре включать счет 20 и отключать 90.02;
- исходные файлы не перезаписывать;
- исправленный маппинг сохранять отдельной копией с суффиксом _v2.

Сделай полный проект с файлами main.py, app/handlers.py, app/services/parsers.py, app/services/audit.py, requirements.txt, README.md и run_bot.ps1.
```

## 5. Файлы, которые должны появиться

| Файл | Что делает |
|---|---|
| `main.py` | запускает бота |
| `app/handlers.py` | отвечает за диалог в Telegram |
| `app/keyboards.py` | кнопки `Аудитор` и `Авто-корректор` |
| `app/states.py` | состояния диалога |
| `app/services/parsers.py` | читает ОСВ и маппинг |
| `app/services/audit.py` | делает проверки и исправления |
| `requirements.txt` | список библиотек |
| `run_bot.ps1` | запуск одной командой |
| `.env.example` | пример настроек |
| `.gitignore` | скрывает секреты и служебные файлы |

## 6. Создать файл `.env`

В папке проекта создайте файл:

```text
.env
```

Внутри должно быть:

```env
BOT_TOKEN=токен_из_BotFather
BOT_WORK_DIR=bot_files
```

Если используется Happ VPN и Python не видит Telegram API, добавьте:

```env
TELEGRAM_PROXY=socks5://127.0.0.1:10808
```

Токен в ответах и скриншотах не показывать.

## 7. Установить зависимости

Откройте PowerShell в папке проекта и выполните:

```powershell
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

Если PowerShell запрещает запуск скриптов, выполните:

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass -Force
```

## 8. Проверить связь с Telegram

Если в проекте есть файл `scripts/check_telegram_connection.py`, выполните:

```powershell
.\.venv\Scripts\python.exe scripts\check_telegram_connection.py
```

Нормальный результат:

```text
Связь с Telegram API есть.
Бот: @имя_бота
```

Если ошибка:

```text
Cannot connect to host api.telegram.org:443
```

значит Python не проходит в Telegram API. Включите VPN или пропишите прокси в `.env`.

Для Happ чаще всего подходит:

```env
TELEGRAM_PROXY=socks5://127.0.0.1:10808
```

## 9. Запустить бота

В PowerShell выполните:

```powershell
.\.venv\Scripts\python.exe main.py
```

Окно не закрывать. Бот работает, пока открыто это окно.

Нормальный запуск:

```text
INFO:aiogram.dispatcher:Start polling
INFO:aiogram.dispatcher:Run polling for bot @имя_бота
```

## 10. Проверить, что бот отвечает

В Telegram откройте своего бота и отправьте:

```text
/start
```

Бот должен ответить:

```text
Выберите режим проверки ОСВ и маппинга P&L.
```

Если есть команда `ping`, можно проверить:

```text
ping
```

Ответ должен быть:

```text
pong
```

## 11. Проверить режим Аудитор

1. Отправьте `/start`.
2. Выберите `Аудитор`.
3. Отправьте файл ОСВ.
4. Отправьте файл `Маппинг_PL`.
5. Бот пришлет текст проверки.

Пример результата:

```text
Результат проверки:
Счет 20, оборот Дт: 0,00 руб.
Счет 20, конечное сальдо Дт: 0,00 руб.

Алерты:
- данных по счету 20 нет.
- аномалия - есть отрицательное конечное сальдо: 76.07.1, 76.07.9.
```

## 12. Проверить режим Авто-корректор

1. Отправьте `/start`.
2. Выберите `Авто-корректор`.
3. Отправьте файл ОСВ.
4. Отправьте файл `Маппинг_PL`.

Если в ОСВ есть оборот Дт по счету `20`, бот должен:

| Счет mask | Что делает |
|---|---|
| `20` | ставит `Активно = 1` |
| `90.02` | ставит `Активно = 0` |

После этого бот отправляет исправленный файл маппинга с суффиксом `_v2`.

Если счета `20` нет, бот напишет:

```text
Авто-корректор ничего не менял: оборота Дт по счету 20 нет.
```

## 13. Какой маппинг нужен

Маппинг — это не выгрузка из 1С. Это шаблон правил для управленческого P&L.

Минимальный шаблон:

| Активно | Статья | Счет mask | Источник | Знак | Сумма |
|---:|---|---|---|---:|---:|
| 1 | Выручка | 90.01 | ОборотКт | 1 |  |
| 1 | Выручка | 90.03 | ОборотДт | -1 |  |
| 0 | Себестоимость | 20 | ОборотДт | -1 |  |
| 1 | Себестоимость | 90.02 | ОборотДт | -1 |  |
| 1 | Коммерческие | 44 | ОборотДт | -1 |  |
| 1 | Общехозяйственные | 26 | ОборотДт | -1 |  |
| 1 | Прочие доходы | 91.01 | ОборотКт | 1 |  |
| 1 | Прочие расходы | 91.02 | ОборотДт | -1 |  |

## 14. Частые ошибки

| Ошибка | Что делать |
|---|---|
| Бот молчит | проверить, запущен ли `python main.py` |
| Нет кнопок | отправить `/start`, кнопки появляются после ответа бота |
| `Cannot connect to host api.telegram.org` | включить VPN или прописать `TELEGRAM_PROXY` |
| `BOT_TOKEN` не найден | проверить файл `.env` |
| `Length mismatch` при маппинге | проверить, что в маппинге есть 6 колонок |
| PowerShell не запускает `.ps1` | выполнить `Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass -Force` |

## 15. Что нельзя делать

| Нельзя | Почему |
|---|---|
| Отправлять токен в чат | это доступ к управлению ботом |
| Перезаписывать исходные файлы ОСВ | нужно сохранить следы ошибок |
| Исправлять ОСВ | ОСВ остается как первичный источник |
| Вбивать суммы руками в расчёты | расчёты должны идти через формулы или код |
