# Как создать бота в Telegram с помощью VS Code

## Результат

После этих шагов у вас будет Telegram-бот, который:

- принимает ОСВ и маппинг;
- считает P_L и BS;
- отправляет Excel-файл с формулами;
- подключает нейросеть через ProxyAPI или OpenAI;
- работает, пока открыто окно PowerShell.

## 1. Что нужно заранее

| Что нужно | Для чего |
|---|---|
| VS Code | открыть и редактировать проект |
| Python | запускать бота |
| Telegram | создать бота и проверить работу |
| Токен Telegram-бота | подключить бота к Telegram |
| Ключ ProxyAPI или OpenAI | подключить нейросеть |

Ключи и токены никому не отправляйте. В чат их не вставляйте.

## 2. Создать бота в Telegram

1. Откройте Telegram.
2. Найдите бота `@BotFather`.
3. Напишите:

```text
/newbot
```

4. Введите название бота.
5. Введите username бота. Он должен заканчиваться на `bot`.
6. BotFather выдаст токен.
7. Сохраните токен только в файл `.env`.

Токен выглядит как длинная строка. Если случайно отправили его в чат, токен нужно сменить у BotFather.

## 3. Открыть проект в VS Code

1. Откройте VS Code.
2. Нажмите `Файл` -> `Открыть папку`.
3. Выберите папку проекта:

```text
C:\Users\Анна\Desktop\тестовая\Создаем бота с нейросетью 6-3_1
```

4. В VS Code откройте терминал:

```text
Терминал -> Создать терминал
```

## 4. Создать виртуальное окружение

В терминале VS Code выполните:

```powershell
python -m venv .venv
```

Если папка `.venv` уже есть, этот шаг можно пропустить.

## 5. Установить зависимости

Выполните:

```powershell
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

Если установка прошла без красных ошибок, всё готово.

## 6. Настроить файл .env

В корне проекта должен быть файл:

```text
.env
```

Откройте его в VS Code и заполните:

```env
BOT_TOKEN=ваш_токен_telegram_бота
BOT_WORK_DIR=bot_files
TELEGRAM_PROXY=socks5://127.0.0.1:1080
OPENAI_API_KEY=ваш_ключ_ProxyAPI_или_OpenAI
OPENAI_BASE_URL=https://api.proxyapi.ru/openai/v1
OPENAI_MODEL=gpt-4o-mini
```

Если Telegram работает без прокси, строку `TELEGRAM_PROXY` можно удалить или закомментировать:

```env
# TELEGRAM_PROXY=socks5://127.0.0.1:1080
```

Если используете обычный OpenAI, строку `OPENAI_BASE_URL` можно удалить.

## 7. Проверить связь с Telegram

Выполните:

```powershell
.\.venv\Scripts\python.exe scripts\check_telegram_connection.py
```

Хороший результат:

```text
Связь с Telegram API есть.
Бот: @имя_вашего_бота
```

Если ошибка, проверьте:

| Ошибка | Что проверить |
|---|---|
| BOT_TOKEN не найден | есть ли `BOT_TOKEN` в `.env` |
| Unauthorized | токен неверный |
| Cannot connect | нужен прокси или нет интернета |

## 8. Проверить логику без Telegram

Выполните:

```powershell
.\.venv\Scripts\python.exe scripts\smoke_check.py
```

Хороший результат:

```text
Smoke-check прошел.
```

## 9. Запустить бота

Выполните:

```powershell
.\.venv\Scripts\python.exe main.py
```

Хороший результат:

```text
Start polling
Run polling for bot @имя_бота
```

Окно PowerShell не закрывайте. Пока окно открыто, бот работает.

## 10. Как пользоваться ботом

1. Откройте Telegram.
2. Найдите своего бота.
3. Напишите:

```text
/start
```

4. Выберите режим.
5. Отправьте файл ОСВ.
6. Отправьте файл маппинга.
7. Получите результат.

Бот должен вернуть:

| Файл или сообщение | Что внутри |
|---|---|
| Текстовая проверка | алерты по ОСВ и маппингу |
| Excel-файл | листы `P_L`, `BS`, `OSV` |
| Ответ нейросети | выводы и что проверить бухгалтеру |

## 11. Как остановить бота

В окне PowerShell нажмите:

```text
Ctrl + C
```

После этого бот перестанет отвечать.

## 12. Как запустить снова

Откройте PowerShell в папке проекта и выполните:

```powershell
.\.venv\Scripts\python.exe main.py
```

Потом в Telegram снова:

```text
/start
```

## 13. Если PowerShell не запускает run_bot.ps1

Если видите ошибку:

```text
Выполнение сценариев отключено в этой системе
```

Не запускайте `run_bot.ps1`. Используйте прямой запуск:

```powershell
.\.venv\Scripts\python.exe main.py
```

## 14. Где лежат результаты

Все загруженные и созданные файлы лежат в папке:

```text
bot_files
```

Примеры файлов:

| Файл | Что это |
|---|---|
| `osv_...xlsx` | загруженная ОСВ |
| `mapping_...xlsx` | загруженный маппинг |
| `financial_report_...xlsx` | готовый отчет P_L и BS |

## 15. Что нельзя делать

- Не отправляйте токены и API-ключи в чат.
- Не удаляйте файлы из `bot_files`, если не уверены.
- Не закрывайте PowerShell, пока бот нужен.
- Не меняйте исходную ОСВ руками ради проверки.

## 16. Как обновлять бота после правок

Если код изменился:

1. Остановите бота:

```text
Ctrl + C
```

2. Установите зависимости:

```powershell
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

3. Запустите снова:

```powershell
.\.venv\Scripts\python.exe main.py
```

## 17. Как должен считаться отчет

`P_L` считается по маппингу:

| Строка | Источник |
|---|---|
| Выручка | счета из маппинга по статье `Выручка` |
| Себестоимость | счета из маппинга по статье `Себестоимость` |
| Коммерческие расходы | счета из маппинга по статье `Коммерческие` |
| Общехозяйственные расходы | счета из маппинга по статье `Общехозяйственные` |
| Прочие доходы | счета из маппинга по статье `Прочие доходы` |
| Прочие расходы | счета из маппинга по статье `Прочие расходы` |

`BS` считается по правилам управленческой группировки:

| Строка | Счета |
|---|---|
| Деньги | 50, 51, 52, 55, 57 |
| Запасы | 10, 41 |
| ДЗ | 60.02, 62.01, 76.09 |
| Основные средства | 02 со знаком минус |
| КЗ | 60.01, 62.02, 76.05 |
| Кредиты и займы | 66, 67 |
| Налоги и взносы | 68, 69 |
| Зарплата | 70 |
| Подотчетные лица | 71 |
| Капитал | 80 |
| Нераспределенная прибыль | 84 |

Если нужна другая управленческая логика, меняется группировка счетов в коде.

## 18. Быстрая команда запуска

Если вы уже всё настроили, каждый раз достаточно:

```powershell
cd "C:\Users\Анна\Desktop\тестовая\Создаем бота с нейросетью 6-3_1"
.\.venv\Scripts\python.exe main.py
```
