# Перенос Cash Flow Lifesaver в n8n Cloud

Дата подготовки: 2026-06-02.

## Результат

Для импорта подготовлен файл:

`cash_flow_lifesaver_n8n_cloud_2026-06-02_v2.json`

Это отдельная копия workflow. Исходный серверный backup не изменен.

Из облачной версии убраны узлы, которым нужен выключенный VPS:

- Supabase;
- загрузка документов в векторную базу;
- триггеры Google Drive для обновления векторной базы.

Сохранены:

- Telegram Trigger;
- ответы AI-агента;
- стартовое сообщение;
- текстовые сообщения;
- голосовые сообщения;
- ссылки на материалы Google Drive;
- сценарии кассового разрыва, прибыли и платежного календаря.

## Импорт

1. Открыть `https://swetlana.app.n8n.cloud`.
2. Войти в учетную запись n8n Cloud.
3. Создать новый workflow через импорт из файла.
4. Выбрать `cash_flow_lifesaver_n8n_cloud_2026-06-02_v2.json`.
5. В узлах Telegram выбрать credential для `@cash_gap_buster_bot`.
6. В узлах `OpenAI Chat Model` и `Transcribe a recording` выбрать OpenAI-совместимый credential ProxyAPI.
7. Сохранить workflow.
8. Нажать `Publish`.
9. Включить workflow.
10. Проверить в Telegram команду `/start`.

## Проверка

Отправить боту сообщения:

1. `кассовый разрыв`
2. `прибыль есть, а денег нет`
3. `как вести платежный календарь`

Затем отправить короткое голосовое сообщение.

## Безопасность

Не отправлять токен Telegram или API-ключ в чат.

В подготовленном JSON секретов нет. Credentials подключаются только внутри n8n Cloud.
