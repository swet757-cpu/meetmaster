# Техническое задание

## Интерактивное коммерческое предложение для продажи услуг финдиректора на аутсорсе

Дата: 29.06.2026

## 1. Результат, который должен получить бизнес

Нужно создать интерактивное коммерческое предложение на основании анкеты собственника и PDF-концепции квиз-лендинга.

Продукт должен не просто задавать вопросы, а подводить клиента к покупке услуги финдиректора на аутсорсе:

| Что делает продукт | Результат для клиента |
|---|---|
| Собирает данные о бизнесе | клиент видит, что предложение учитывает его ситуацию |
| Выявляет финансовые риски | клиент понимает, где теряет деньги и контроль |
| Показывает выводы 5 экспертных логик | результат выглядит как профессиональный разбор |
| Формирует персональное КП | клиент получает понятное предложение по услуге |
| Дает быстрые действия | результат можно скопировать, сохранить в PDF, отправить в Telegram или по почте |

## 6. Структура диагностики

Анкета в DOCX содержит 50 вопросов. Для коммерческого предложения нужна сокращенная версия: 24-30 вопросов.

Полную анкету использовать как методическую базу. В интерфейсе показывать только вопросы, которые влияют на продажу и персонализацию КП.

| Блок | Вопросы из анкеты | Что берем в интерактивное КП |
|---|---:|---|
| Профиль бизнеса | 1-6 | сфера, выручка, юрлица, направления, сотрудники |
| Деньги | 7-12 | остаток денег, кассовые разрывы, платежный календарь |
| Прибыль | 13-18 | прибыль, направления, рентабельность, дивиденды |
| Отчеты | 19-26 | ДДС, ОПиУ, баланс, план-факт, дашборд |
| Боли | 27-38 | хаос, налоги, долги, смешение денег, поздние данные |
| Решения | 39-44 | как собственник принимает финансовые решения |
| Готовность | 45-50 | данные, ответственный, срок старта, приоритет |

## 8. Правила расчета баллов

Баллы риска считаются только по вопросам, где есть финансовый риск.

Формула:

```text
Баллы риска = сумма баллов по выбранным ответам
```

Для множественного выбора по болям:

```text
Баллы по болям = количество выбранных болей × 2
```

Для вопроса «Какие отчеты есть сейчас»:

| Ответ | Баллы |
|---|---:|
| ничего | 6 |
| только бухгалтерские отчеты | 4 |
| 1-2 управленческих отчета | 2 |
| 3-4 управленческих отчета | 1 |
| 5 и более отчетов | 0 |

## 9. Интерпретация риска

| Баллы | Уровень | Вывод |
|---:|---|---|
| 0-15 | Низкий риск | учет частично выстроен, нужна проверка качества отчетов |
| 16-35 | Средний риск | есть системные пробелы, нужны ДДС и платежный календарь |
| 36-55 | Высокий риск | высокий риск кассовых разрывов и управленческих ошибок |
| 56+ | Критический риск | финансы управляются хаотично, нужен запуск регулярного финансового контура |

## 10. Расчет ориентира потерь

Расчет не должен подаваться как точная бухгалтерская оценка. Это ориентир для коммерческого предложения.

Формула:

```text
Ориентир потерь = месячная выручка × коэффициент риска
```

| Уровень риска | Коэффициент риска |
|---|---:|
| Низкий риск | 1% |
| Средний риск | 3% |
| Высокий риск | 5% |
| Критический риск | 8% |

Пример логики:

```text
Выручка 5 000 000 руб. × 3% = 150 000 руб.
```

Если выручка не указана, выводить:

```text
данных нет
```

| Выявленная потребность | Что предложить |

## 12. Консилиум из 5 экспертных логик

Финальный результат должен выглядеть как разбор с разных сторон.

| Эксперт | Что анализирует | Пример вывода |
|---|---|---|
| Финансовый директор | деньги, прибыль, дивиденды, управленческие решения | бизнесу не хватает регулярного финансового контура |
| Методолог управленческого учета | ДДС, ОПиУ, баланс, план-факт, дашборд | нужно собрать связку отчетов, а не отдельные таблицы |
| Налоговый эксперт | обязательные платежи, налоговый календарь, внезапные суммы | налоги должны попадать в платежный календарь заранее |
| Риск-аналитик | кассовые разрывы, долги, дебиторка, кредиторка | главный риск в ближайшие 2-4 недели — нехватка денег |
| Коммерческий эксперт | готовность к покупке, срок старта, приоритет | клиенту лучше предложить формат, который закрывает ближайшую боль |

Каждый эксперт должен выводить:

| Поле | Описание |
|---|---|
| Оценка | от 1 до 10 |
| Диагноз | короткий вывод |
| Рекомендация | что делать первым шагом |

## 13. Правило выбора формата услуги

| Условия | Формат услуги |
|---|---|
| низкий риск, отчеты частично есть | разовая диагностика управленческого учета |
| средний риск, нет регулярности | запуск управленческого учета |
| высокий риск, есть долги или кассовые разрывы | финдиректор на аутсорсе |
| критический риск, хаос в деньгах и отчетах | восстановление данных + финдиректор на аутсорсе |

## 18. Критерии приемки

| Проверка | Ожидаемый результат |
|---|---|
| Пользователь проходит диагностику | вопросы переключаются без перезагрузки страницы |
| Баллы считаются | сумма меняется по выбранным ответам |
| Формула потерь работает | сначала формула, потом результат |
| Нет выручки | выводится «данных нет» |
| Рекомендации персонализируются | вывод зависит от ответов |
| Экспертные выводы разные | 5 логик дают разные оценки |
| Кнопка копирования работает | текст КП попадает в буфер обмена |
| Печать работает | сохраняется финальное КП без лишних экранов анкеты |
| Telegram работает | открывается отправка результата |
| Почта работает | создается письмо с темой и текстом |
| Мобильная версия | текст и кнопки не перекрываются |
