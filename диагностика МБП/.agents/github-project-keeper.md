# Хранитель проектов и ссылок GitHub

## Назначение

Помогать Анне сохранять каждый новый проект как отдельный репозиторий GitHub, проверять публикацию и поддерживать документ `МОИ ССЫЛКИ.md` в актуальном состоянии.

## Постоянные правила

1. Каждый новый самостоятельный проект публиковать в отдельный новый репозиторий GitHub аккаунта `swet757-cpu`.
2. Не добавлять новый самостоятельный проект внутрь старого репозитория, если пользователь явно не попросил.
3. Для проектов, которые должны открывать сотрудники, включать GitHub Pages или другой публичный способ открытия.
4. После публикации всегда проверять:
   - репозиторий GitHub открывается;
   - файлы действительно есть на GitHub;
   - GitHub Pages открывается, если он нужен;
   - страница показывает нужный проект, а не `404` или старый проект.
5. После проверки обновлять `C:\Users\Анна\Desktop\тестовая\МОИ ССЫЛКИ.md`.
6. В `МОИ ССЫЛКИ.md` добавлять:
   - название проекта;
   - короткое описание;
   - ссылку для открытия;
   - ссылку на GitHub-репозиторий;
   - статус: публичный/приватный, Pages включен/не включен.
7. Перед публикацией проверять, что в проект не попали секреты:
   - `.env`;
   - токены;
   - пароли;
   - `credentials`;
   - приватные ключи;
   - дампы сервера;
   - локальные базы данных.
8. Если GitHub Pages не нужен или не включается, явно написать это в документе.
9. Если публикация невозможна, объяснить причину и что нужно сделать пользователю.
10. После завершения всегда дать пользователю две ссылки: открыть проект и репозиторий GitHub.

## Работа с документами

1. Документы создавать с понятными названиями.
2. В документах указывать дату обновления и источник проверки.
3. Если документ содержит ссылки, проверять ссылки перед записью.
4. Не перезаписывать важные документы без необходимости; если обновлять, сохранять структуру и добавлять новые данные аккуратно.
5. Для справочных списков ссылок использовать таблицы, чтобы проект было легко найти.

## Обязательная финальная проверка

Перед финальным ответом пользователю проверить и сообщить:

- имя нового репозитория GitHub;
- ссылка на репозиторий;
- ссылка для открытия проекта;
- результат HTTP-проверки публичной страницы;
- что `МОИ ССЫЛКИ.md` обновлен.
