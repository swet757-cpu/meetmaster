# Техническое задание v1

## Проект: MeetMaster

## Executive Summary

MeetMaster - Telegram-бот для согласования встреч через Google Календарь.

Пользователь открывает бота, выбирает свободный слот, указывает email и описание встречи. Заявка уходит администратору. После подтверждения встреча создается в Google Календаре, пользователь получает уведомление.

Встречи доступны только с понедельника по пятницу, с 09:00 до 17:00. Минимальный срок записи - за 1 день. Буфер между встречами - 15 минут. Пользователь может отменить встречу не позже чем за 24 часа.

MVP делается как Telegram-чат-бот. MiniApp, веб-кабинет, CRM и экспорт в Excel остаются на будущие версии.

## 4. Пользовательские сценарии

### 4.1. Запись на встречу

1. Пользователь открывает Telegram-бота.
2. Бот показывает основное меню.
3. Пользователь выбирает действие "Записаться на встречу".
4. Бот показывает доступные даты.
5. Пользователь выбирает дату.
6. Бот предлагает длительность встречи: 30, 45 или 60 минут.
7. Пользователь выбирает длительность.
8. Бот показывает доступные слоты на выбранную дату.
9. Пользователь выбирает слот.
10. Бот запрашивает email.
11. Пользователь вводит email.
12. Бот проверяет формат email.
13. Бот запрашивает тему или описание встречи.
14. Пользователь вводит описание.
15. Бот показывает итог заявки.
16. Пользователь подтверждает отправку заявки.
17. Заявка получает статус "На согласовании".
18. Слот временно блокируется внутри бота.
19. Администратор получает заявку.
20. Пользователь получает уведомление после решения администратора.

### 4.2. Просмотр своих заявок

1. Пользователь выбирает действие "Мои заявки".
2. Бот показывает активные заявки и подтвержденные встречи.
3. Для каждой заявки отображаются дата, время, длительность, статус.

### 4.3. Отмена пользователем

1. Пользователь открывает список своих заявок.
2. Пользователь выбирает встречу.
3. Если до встречи осталось не меньше 24 часов, бот предлагает отменить встречу.
4. Пользователь подтверждает отмену.
5. Встреча получает статус "Отменена".
6. Если событие уже создано в Google Календаре, оно отменяется или удаляется.
7. Администратор получает уведомление об отмене.

Если до встречи осталось меньше 24 часов, отмена через бота недоступна. Бот показывает сообщение о необходимости связаться с администратором.

### 4.4. Помощь

1. Пользователь выбирает действие "Помощь".
2. Бот показывает текст: "Связаться с администратором".

## 5. Админ-сценарии

### 5.1. Согласование заявки

1. Администратор получает новую заявку в Telegram.
2. В заявке отображаются:
   - имя пользователя из Telegram;
   - фамилия пользователя из Telegram, если доступна;
   - username, если доступен;
   - email;
   - дата встречи;
   - время начала;
   - длительность;
   - описание встречи.
3. Администратор выбирает действие:
   - подтвердить;
   - отклонить;
   - перенести;
   - отменить.
4. При подтверждении бот повторно проверяет свободность слота в Google Календаре.
5. Если слот свободен, бот создает событие в Google Календаре.
6. Пользователь получает уведомление о подтверждении.
7. Если слот уже занят, администратор получает сообщение об ошибке и предложение перенести встречу.

### 5.2. Отклонение заявки

1. Администратор выбирает действие "Отклонить".
2. Заявка получает статус "Отклонена".
3. Временная блокировка слота снимается.
4. Пользователь получает уведомление об отклонении.

### 5.3. Перенос встречи

1. Администратор выбирает действие "Перенести".
2. Администратор выбирает новую дату и время.
3. Бот проверяет доступность нового слота.
4. Заявка получает статус "Перенесена" или "Подтверждена", если встреча сразу утверждена.
5. Если событие уже было создано в Google Календаре, оно обновляется.
6. Пользователь получает уведомление с новой датой и временем.

### 5.4. Отмена встречи администратором

1. Администратор выбирает заявку или встречу.
2. Администратор нажимает "Отменить".
3. Встреча получает статус "Отменена".
4. Событие в Google Календаре отменяется или удаляется.
5. Пользователь получает уведомление.

### 5.5. Управление закрытыми днями

1. Администратор открывает раздел настроек.
2. Администратор выбирает действие "Закрыть день".
3. Администратор указывает дату.
4. При необходимости администратор указывает причину.
5. Дата становится недоступной для пользователей.

Администратор должен иметь возможность открыть ранее закрытый день.

## 8. Интеграции

| Интеграция | Назначение |
|---|---|
| Telegram Bot API | работа чат-бота |
| Google Calendar API | проверка занятости и создание встреч |
| Google OAuth | подключение календаря администратора |
| PostgreSQL | хранение пользователей, заявок, настроек и закрытых дней |
| Telegram MiniApp | v2, удобный интерфейс выбора слотов |

### Google Календарь

В MVP используется один Google Календарь администратора.

Бот должен:

1. Получать занятость календаря.
2. Исключать занятые интервалы из доступных слотов.
3. Создавать событие после подтверждения заявки.
4. Добавлять email пользователя как участника встречи.
5. Обновлять событие при переносе.
6. Отменять или удалять событие при отмене встречи.

## 11. Ошибки и исключительные сценарии

| Ситуация | Поведение |
|---|---|
| Google Calendar недоступен | бот сообщает, что запись временно недоступна |
